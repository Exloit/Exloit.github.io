跨域方案枚举

Posted on 2018-03-24

JS跨域问题重理解

为什么跨域会出现问题？因为浏览器设定的同源策略是的JS文件不能随心所欲的从任意网站读取数据。一定程度上保证了数据的安全性。

跨域的问题是跨域前端和被跨域后端的问题。
如果需要进行跨域数据传输。那么可以是用一下方法

0x01 使用跨域资源共享（CORS）

CORS定义了浏览器域服务器应该如何沟通。CORS背后的基本思想就是使用自定义的HTTP头部让浏览器与服务器进行沟通。
如渗透中常常见到的： Access-Control-Allow-Origin: *

0x02 使用JSONP

JSONP 学名 填充式JSON。
JSONP名字有些抽象，但是其实是JSON对象的一种用法。抽象的概括一下就是被包含在函数调用中的JSON。
callback({“name”,”name1”});
由上行代码可看出JSONP由两部分组成:回调函数和JSON数据。
回调函数是在响应到来的时候页面执行的函数。
而数据就是回调函数中的JSON数据

原理：
数据返回后会被相应函数执行。
缺点：只能GET，无法解决不同域的连个页面之间的javascript调用问题。

0x03 document.domain + iframe( iframe 用来ajax通信)

首先架设一个网站中有一个iframe中包含不同域的页面。根据同源策略，主页面的 JS不可以获取iframe页面中的数据的。
这个时候，我们可以设置document.domain为相同的值。但是这个有个限制。document.domain只能设置本身和更高级域名。
这就要求两个页面的主域名必须相同。然后通过ajax正常交流数据

0x04 使用window.name + iframe

windows对象的name属性。该属性有个特征：在一个窗口的生命周期内，窗口所载入的所有页面都是属于一个windows.name的。每个页面
窗口载入的所有页面都是共享一个windows.name的。每个页面对windows.name都有读写权限。windows.name是持久存在一个窗口载入过的所有页面中的。
这个我自己也不是很理解。感觉上是通过windows.name这个来传递数据

0x05 使用Windows.postMessage方法

HTML5引进了新方法postMessage。可以在不同的windows下传递数据不受域的影响。目前IE8+,Firefox，chrome,Opera等浏览器都支持该方法。

windows.postMessage(message,targetOrigin)

首先调用这个方法的windows对象是消息对象。message为要发送的信息，类型只能为字符串。下一个参数是限定接受消息的对象所在的域
。可以使用通配符。
需要接收消息的window对象，可通过监听自身的message事件来获取传过来的消息，消息内容储存在该事件对象的data属性中。

这个我的理解是远程对象调用postMeassge()并设置好接受对象域。接受对象域就可已接受到内容。

0x06 flash跨域

在需要跨域的服务器上吗设置crossdomain.xml.这个服务就被配置成为可以被其它域跨域访问的服务器了。
crossdomain.xml文件，该文件限制了flash是否可以跨域读写数据以及允许从什么地方跨域读写数据。

免杀Powershell

Posted on 2018-03-24

免杀Powershell

这个方法是看到先知社区有人写出对ps脚本的混淆的方法绕过windows defender , 文章中提到windows defender对ps脚本的查杀能力很強，可以通过混淆的方法绕过它。但是这种方法是有缺陷的，就是文章所提到的是windows defender对静态文本形势的脚本文件的查杀。而不是可执行文件的查杀。那么我们来想想可执行文件的查杀，应该怎么做？
很简单，其实ps脚本可以使用.net动态编译成可执行程序。
原理是利用将PS脚本嵌入一个.net程序，然后使用调用powershell运行。本来经过混淆的脚本可以通过杀软的静态查杀了，现在还使用这种方法。杀软仍然无法判断这个.net程序是否是一个恶意程序 。

但是在virustotals测试时，有个叫安博士的杀软没有绕过。其它主流杀软都无法判定。

Metasploit 自动化

0x00 前言

利用msf 实现自动化渗透攻击，已经不是什么难事了，最近打算实现自动化脚本的msf这一块部分，让前面的自动化脚本和新的自动化脚本结合，减少工作量。国内的大部分文档作者不是利用sh实现自动化，就是利用spiderlad写的那个简单版的msfrpc脚本，坑点巨多，代码巨丑。简直不忍直视。我在github上搜寻了一天找到3个相对靠谱的项目。

  • https://github.com/SpiderLabs/msfrpc 最简单的python rpc实现，不够好

  • https://github.com/RayofLightz/pymsfrpc 巨坑，我第一次选择了这个做二次开发，结果遇到找不到api call的错误，直接导致前功尽弃

  • https://github.com/allfro/pymetasploit 最好的方案

0x01 pymetasploit 的使用方法

pymestaploit是完全成熟的msfrpc的python库代码。它不像上文提到的另外两个库文件仅仅包含了部分的rpc.call，而是包含了完整的msfrpc.call。原先没有选择这个库的原因是它的更新时间是3年前，就是15年初更新的。一直没有再变化过。我们目前参考的官方文档RemoteAPI_4.1.pdf。都还是2011年版本的呢，对比代码提交时间，这个库应该是很新的了。

使用前需要打开msfrpc服务，作者推荐使用msfrpcd脚本开启服务

kali下的该文件路劲是：

/usr/share/metasploit-framework

进入目录运行：

./msfrpcd -P mypassword -n -f -a 127.0.0.1

即可打开服务端口

库文件使用：

首先需要创建一个client

client = MsfRpcClient('mypassword')

client对象中包含core,modules,sessions,jobs,consoles,authenticated,plugins,db,auth,login,logout等函数。

这些函数会调用后会创建相应的类，这些类中包含了一些对应api的操作方法。

本文中重点介绍console类，和session类这个是自动化操作中主要用到的两个大类

0x02 console类

在client中调用consoles函数，会调用ConsoleManager函数创建ConsoleManager类。

利用ConsoleManager类的console方法可以创建MsfConsole类

所以创建Console的代码如下:

console0 = client.consoles.console()

这时，我们拥有了一个MsfConsole类。类似于一个命令行窗口，我们可以对这个窗口写入，读取数据来执行命令。

该类的标准方法包含如下：

  • read() 读取运行结果

  • write(command) 写入运行命令

  • sessionkill()杀死所有活动会话

  • sessiondetack() 退出当前会话

  • tabs(line) 补齐命令

  • destroy() 销毁窗口（当前端口）

0x03 session 类

同理 ，在client中调用sessions函数，会创建SessionManager类。SessionManager类中有一个session函数和list函数。list函数可以列出当前活动的会话有哪些，并且告诉用户会话的ID值是多少（ID值是从1开始的）

我们将ID值传入session()函数，函数会根据session的类型创建类，MeterpreterSession和ShellSession。

我一般使用的是MeterpreterSession。

session = client.sessions.session(1)

MeterpreterSession的可执行方法有：

  • read() 读取命令执行结果

  • write(data) 写入需要执行的命令

  • rungingle(data)运行某个meterpreter命令

  • runscript(path) 运行某个脚本

  • sep() 设置操作系统路径分隔符？

  • detach()

  • kill()

  • tabs()

使用GPP漏洞进行域渗透

Posted on 2018-03-04

使用Windows 2008 GPP漏洞进行域渗透

0x00 GPP简介

GPP(组策略首选项)是windows 在server 2008操作系统中映入的客户端拓展插件。它可以完成很多组策略无法进行的系统及应用配置。GPP通过操作组策略对象GPO对域中的资源进行管理。
GPP可以指定某一个域账户为所有计算机的本地管理账户，但是这个帐号信息存在\[Domain Controller]\SYSVOL[Domain]\Policies的Grouop.xml中的cpassword。cpassword使用的是AES加密值。但是在AD中所有的用户都可以读取Group.xml。微软公司又曾经公布了AES的密钥。可以用来解密这种密码。这样就造成了很大的安全风险。

##0x01 利用

这个漏洞比较老了，今天在一个渗透框架里看到，不太理解什么意思，就花了点时间学习了下，网络上已经有很多公开的ps脚本可以利用这个漏洞。关键词Invoke-GPPpassword

##0x02 防御
重点应该记录一下防御方式，目前针对GPP漏洞的防御方式主要有：

  • 打补丁(KB2962486),这个补丁要安装在所有使用了RSAT的系统上。
  • 删除现有的GPP里包含的密码的XML文件
  • 使用LAPS更改本地Administrtor密码。

Content-Security-Policy-Introduction

Posted on 2018-03-03

#Web安全之内容安全策略学习

内容安全策略（简称CSP），网站通过发送一个 CSP 头部，来告诉浏览器什么是被授权执行的与什么是需要被禁止的。用于帮助缓解跨站脚本攻击和数据注入攻击上有很好的效果。大部分浏览器如今都支持CSP，并且使用的是CSP1.0标准中的Content-Security-Policy头部和其它相关指令。

CSP基础知识

作为一种安全策略，CSP由两种干事指定：HTTP HEADRE 和 HTML meta标签。

CSP策略又分为两类：Content-Security-Policy 和 Content-Security-Policy-Report-Only

HTTP Content-Security-Policy 头可以指定一个或多个资源是安全的，而Content-Security-Policy-Report-Only则是允许服务器检查（非强制）一个策略。多个头的策略定义由优先采用最先定义的。

ubuntu 安装 metasploit

metasploit 是一款老牌的漏洞利用工具了。以前总是在windows上面安装它。或者直接使用kali的版本。没想过在linux上面安装。今天有需求，就试着安装了 一下。比windows下面使用的爽多了。
下面是安装步骤

  1. 打开linux console终端
  2. 下载安装程序：
    $ wget http://downloads.metasploit.com/data/releases/metasploit-latest-linux-x64-installer.run
    根据自己的版本选择x64还是x32
  3. 添加文件的执行的权限
    user@ubuntu:~4 chmod +x ./metasploit-latest-linux-x64-installer.run
  4. 运行安装程序
    sudo ./metasploit-latest-linux-x64-installer.run
  5. 然后一直回车，中间有个Y要输入。

  6. 不要忘记注册一个用户！API需要用户登录
