Metasploit 自动化

0x00 前言

利用msf 实现自动化渗透攻击，已经不是什么难事了，最近打算实现自动化脚本的msf这一块部分，让前面的自动化脚本和新的自动化脚本结合，减少工作量。国内的大部分文档作者不是利用sh实现自动化，就是利用spiderlad写的那个简单版的msfrpc脚本，坑点巨多，代码巨丑。简直不忍直视。我在github上搜寻了一天找到3个相对靠谱的项目。

  • https://github.com/SpiderLabs/msfrpc 最简单的python rpc实现，不够好

  • https://github.com/RayofLightz/pymsfrpc 巨坑，我第一次选择了这个做二次开发，结果遇到找不到api call的错误，直接导致前功尽弃

  • https://github.com/allfro/pymetasploit 最好的方案

0x01 pymetasploit 的使用方法

pymestaploit是完全成熟的msfrpc的python库代码。它不像上文提到的另外两个库文件仅仅包含了部分的rpc.call，而是包含了完整的msfrpc.call。原先没有选择这个库的原因是它的更新时间是3年前，就是15年初更新的。一直没有再变化过。我们目前参考的官方文档RemoteAPI_4.1.pdf。都还是2011年版本的呢，对比代码提交时间，这个库应该是很新的了。

使用前需要打开msfrpc服务，作者推荐使用msfrpcd脚本开启服务

kali下的该文件路劲是：

/usr/share/metasploit-framework

进入目录运行：

./msfrpcd -P mypassword -n -f -a 127.0.0.1

即可打开服务端口

库文件使用：

首先需要创建一个client

client = MsfRpcClient('mypassword')

client对象中包含core,modules,sessions,jobs,consoles,authenticated,plugins,db,auth,login,logout等函数。

这些函数会调用后会创建相应的类，这些类中包含了一些对应api的操作方法。

本文中重点介绍console类，和session类这个是自动化操作中主要用到的两个大类

0x02 console类

在client中调用consoles函数，会调用ConsoleManager函数创建ConsoleManager类。

利用ConsoleManager类的console方法可以创建MsfConsole类

所以创建Console的代码如下:

console0 = client.consoles.console()

这时，我们拥有了一个MsfConsole类。类似于一个命令行窗口，我们可以对这个窗口写入，读取数据来执行命令。

该类的标准方法包含如下：

  • read() 读取运行结果

  • write(command) 写入运行命令

  • sessionkill()杀死所有活动会话

  • sessiondetack() 退出当前会话

  • tabs(line) 补齐命令

  • destroy() 销毁窗口（当前端口）

0x03 session 类

同理 ，在client中调用sessions函数，会创建SessionManager类。SessionManager类中有一个session函数和list函数。list函数可以列出当前活动的会话有哪些，并且告诉用户会话的ID值是多少（ID值是从1开始的）

我们将ID值传入session()函数，函数会根据session的类型创建类，MeterpreterSession和ShellSession。

我一般使用的是MeterpreterSession。

session = client.sessions.session(1)

MeterpreterSession的可执行方法有：

  • read() 读取命令执行结果

  • write(data) 写入需要执行的命令

  • rungingle(data)运行某个meterpreter命令

  • runscript(path) 运行某个脚本

  • sep() 设置操作系统路径分隔符？

  • detach()

  • kill()

  • tabs()
