Content-Security-Policy-Introduction

#Web安全之内容安全策略学习

内容安全策略（简称CSP），网站通过发送一个 CSP 头部，来告诉浏览器什么是被授权执行的与什么是需要被禁止的。用于帮助缓解跨站脚本攻击和数据注入攻击上有很好的效果。大部分浏览器如今都支持CSP，并且使用的是CSP1.0标准中的Content-Security-Policy头部和其它相关指令。

CSP基础知识

作为一种安全策略，CSP由两种干事指定：HTTP HEADRE 和 HTML meta标签。

CSP策略又分为两类：Content-Security-Policy 和 Content-Security-Policy-Report-Only

HTTP Content-Security-Policy 头可以指定一个或多个资源是安全的，而Content-Security-Policy-Report-Only则是允许服务器检查（非强制）一个策略。多个头的策略定义由优先采用最先定义的。
